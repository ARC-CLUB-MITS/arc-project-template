## Git Workflow

Branch usage:

```
main      → stable version 
staging   → integration and testing
feature/* → task-specific development
```

Rules:

* No direct commits to `main`
* All changes via pull requests
* Feature branches must be scoped and short-lived

---

## Contribution Guidelines

* One task = one feature branch
* Commit frequently with clear messages
* Push code regularly (no local-only work)
* Inactive tasks will be reassigned

Execution > discussion.
